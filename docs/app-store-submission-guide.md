# LabStock App Store リリース手順ガイド

このドキュメントは、LabStockアプリをApp Storeに公開するための全手順をまとめたものです。ビルドは既に完了しており（ビルド番号6、IPA取得済み）、以下のステップに従ってTestFlight配布とApp Store審査提出を進めます。

---

## ステップ1: IPAファイルのダウンロード

EAS Buildで生成されたIPAファイルを、以下のURLからダウンロードしてください。

| 項目 | 内容 |
|------|------|
| IPA URL | https://expo.dev/artifacts/eas/voUiZXvTfxU6fw4Csn35ZN.ipa |
| ビルド番号 | 6 |
| バージョン | 1.0.0 |

ブラウザでURLにアクセスすると、IPAファイルが自動的にダウンロードされます。このファイルをMac上の任意のフォルダに保存してください。

---

## ステップ2: Transporter でIPAをアップロード

Apple公式の **Transporter** アプリを使用して、IPAファイルをApp Store Connectにアップロードします。

1. Mac App Storeから **Transporter** をダウンロード・インストールします（無料）。
2. Transporterを起動し、Apple Developer アカウント（Apple ID）でサインインします。
3. ダウンロードしたIPAファイルをTransporterのウィンドウにドラッグ＆ドロップします。
4. 「配信」ボタンをクリックしてアップロードを開始します。
5. アップロードが完了すると、App Store Connectの「TestFlight」タブにビルドが表示されます（反映まで数分〜30分程度かかる場合があります）。

> **注意**: Transporterの代わりに、Xcodeの「Organizer」からもアップロード可能です。ただし、EAS Buildで生成したIPAの場合はTransporterの方が簡便です。

## ステップ3: App Store Connect でアプリを登録

[App Store Connect](https://appstoreconnect.apple.com/) にサインインし、新しいアプリを登録します。

### 3-1. 新しいアプリの作成

1. App Store Connectのダッシュボードで「マイApp」をクリックします。
2. 左上の「+」ボタンから「新規App」を選択します。
3. 以下の情報を入力します。

| 項目 | 入力値 |
|------|--------|
| プラットフォーム | iOS |
| 名前 | LabStock - 研究室在庫管理 |
| プライマリ言語 | 日本語 |
| バンドルID | space.manus.labstock.t20260305040342（ドロップダウンから選択） |
| SKU | labstock-ios-001 |

4. 「作成」をクリックします。

### 3-2. アプリ情報の設定

「App情報」タブで以下を設定します。

| 項目 | 設定値 |
|------|--------|
| サブタイトル | 備品・薬品の在庫をスマートに管理 |
| カテゴリ（プライマリ） | ビジネス |
| カテゴリ（セカンダリ） | 仕事効率化 |
| コンテンツの権利 | 「このAppにはサードパーティのコンテンツは含まれていません」を選択 |
| 年齢レーティング | 質問に回答（すべて「なし」で4+になります） |

### 3-3. 価格と配信地域

「価格および配信状況」タブで以下を設定します。

| 項目 | 設定値 |
|------|--------|
| 価格 | 無料 |
| 配信地域 | 日本（必要に応じて他の国も追加） |

## ステップ4: バージョン情報の入力

「iOS App」セクションの「1.0 提出準備中」ページで、以下の情報を入力します。

### 4-1. スクリーンショット

スクリーンショットは実機で撮影するか、シミュレータで撮影したものを使用します。以下の2つのサイズが必須です。

| デバイスサイズ | 解像度（ポートレート） |
|---------------|----------------------|
| 6.9インチディスプレイ | 1260 x 2736 px |
| 6.5インチディスプレイ | 1284 x 2778 px |

推奨するスクリーンショット構成は以下の5枚です。

| 番号 | 画面 | 説明 |
|------|------|------|
| 1 | 在庫一覧 | メイン画面。検索バー、カテゴリフィルター、物品カード一覧 |
| 2 | 物品追加 | 登録フォーム。企業名、型番、個数、画像添付 |
| 3 | アラート | 在庫不足アラート一覧。閾値以下の物品表示 |
| 4 | グループ | グループ一覧、メンバー管理、共有在庫 |
| 5 | 設定 | カテゴリ管理、ダークモード、エクスポート |

> **スクリーンショットの撮影方法**: TestFlightでアプリをインストール後、実機で各画面を開いてスクリーンショットを撮影するのが最も確実です。iPhoneの「設定 > 一般 > 情報」でデバイスの画面サイズを確認し、適切な解像度のスクリーンショットを取得してください。

### 4-2. 説明文

**プロモーションテキスト**（170文字以内）:

```
研究室の備品・薬品の在庫管理をスマートに。バーコードスキャン、グループ共有、在庫アラートで、チーム全体の在庫状況をリアルタイムに把握できます。
```

**説明**（4000文字以内）:

```
LabStockは、大学・企業の研究室向けに設計された在庫管理アプリです。備品、薬品、消耗品などの在庫を効率的に管理し、チームメンバーと共有できます。

【主な機能】

■ 在庫管理
・物品名、企業名、型番、個数、画像を登録
・バーコード/QRコードスキャンで素早く物品を追加
・カテゴリタグで物品を分類（複数タグ対応）
・カスタムカテゴリの作成も可能

■ 在庫アラート
・物品ごとに在庫閾値を設定
・閾値を下回ると自動でアラート通知
・アラート一覧で不足物品を一目で確認

■ グループ機能
・グループを作成し、招待リンクでメンバーを招待
・グループ内で在庫情報をリアルタイム共有
・変更ログで誰がいつ何を変更したか追跡
・データの秘匿性を保護

■ データ管理
・Excel形式での在庫データエクスポート
・検索・ソート機能で素早く物品を検索
・ダークモード対応

研究室の在庫管理を紙やExcelから卒業しませんか？LabStockで、チーム全体の在庫状況をスマートに管理しましょう。
```

## ステップ5: TestFlight でテスト配布

審査提出前に、TestFlightで実機テストを行うことを推奨します。

1. App Store Connectの「TestFlight」タブを開きます。
2. アップロードしたビルドが表示されていることを確認します。
3. 「輸出コンプライアンス情報」の質問に回答します（暗号化は「いいえ」を選択 — 標準的なHTTPS通信のみ使用のため）。
4. 「内部テスター」グループにご自身を追加します。
5. TestFlightアプリ（iPhoneにインストール）でビルドをダウンロードしてテストします。

### テスト確認項目

以下の項目を実機で確認してください。

| 確認項目 | 確認内容 |
|---------|---------|
| 起動 | アプリが正常に起動し、スプラッシュ画面が表示されること |
| 在庫一覧 | 物品の追加・編集・削除が正常に動作すること |
| バーコードスキャン | カメラが起動し、バーコード読み取りが機能すること |
| アラート | 閾値設定とアラート表示が正常に動作すること |
| グループ機能 | グループ作成・招待・在庫共有が動作すること |
| 広告表示 | AdMobバナー広告が正常に表示されること |
| ダークモード | ダークモード切替が全画面で正しく反映されること |
| Excel出力 | 在庫データのエクスポートが正常に動作すること |

## ステップ6: 審査に提出

TestFlightでの確認が完了したら、App Store審査に提出します。

1. App Store Connectの「iOS App」セクションに戻ります。
2. すべての必須項目（スクリーンショット、説明文、ビルド選択など）が入力されていることを確認します。
3. 右上の「審査に追加」ボタンをクリックします。
4. 確認ダイアログで「送信」をクリックします。

審査は通常24〜48時間で完了しますが、初回提出の場合はそれ以上かかることもあります。審査結果はメールとApp Store Connectの通知で届きます。

---

## ステップ7: リリース

審査が承認されると、以下の選択肢があります。

| リリース方法 | 説明 |
|-------------|------|
| 手動リリース | 承認後、自分のタイミングでリリースボタンを押す |
| 自動リリース | 承認と同時に自動的にApp Storeに公開される |

初回リリースの場合は「手動リリース」を選択し、公開タイミングをコントロールすることを推奨します。

## トラブルシューティング

### ビルドがApp Store Connectに表示されない場合

Transporterでアップロード後、ビルドの処理に時間がかかることがあります。通常は30分以内に表示されますが、最大で数時間かかる場合もあります。しばらく待ってからページを更新してください。

### 審査でリジェクトされた場合

リジェクト理由を確認し、指摘事項を修正してから再提出します。よくあるリジェクト理由は以下の通りです。

| リジェクト理由 | 対処法 |
|---------------|--------|
| メタデータの不備 | 説明文やスクリーンショットを修正 |
| バグの存在 | 指摘されたバグを修正して新ビルドをアップロード |
| プライバシーポリシーの不備 | プライバシーポリシーの内容を更新 |
| ガイドライン違反 | Apple Human Interface Guidelinesに準拠するよう修正 |
