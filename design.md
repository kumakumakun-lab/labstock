# LabStock - UI設計書

## 1. アプリ概要

研究室の備品・薬品の在庫管理に特化したiOSモバイルアプリケーション。シンプルで直感的な操作性を重視し、片手操作を前提としたデザインとする。

## 3. 画面一覧

### 3.1. タブ構成（3タブ）

| タブ | アイコン | 画面名 | 説明 |
|:---|:---|:---|:---|
| 在庫一覧 | list.bullet (view-list) | InventoryList | メイン画面。全物品の一覧表示 |
| 追加 | plus.circle.fill (add-circle) | AddItem | 新規物品登録フォーム |
| アラート | bell.fill (notifications) | Alerts | 在庫不足アラート一覧 |

### 3.2. 各画面の詳細

#### 在庫一覧画面（InventoryList）
- **ヘッダー**: アプリタイトル「LabStock」
- **検索バー**: 物品名・企業名・型番で検索可能
- **ソートボタン**: 名前順 / 個数順 / 登録日順
- **物品カード**: 画像サムネイル（左）、物品名・企業名・型番（中央）、個数バッジ（右）
- **個数バッジの色分け**: 十分（success）、少量（warning）、不足（error）
- **タップ操作**: カードタップで詳細画面へ遷移
- **画面最下部**: バナー広告表示エリア

#### 物品詳細画面（ItemDetail）- モーダル表示
- **画像表示**: 大きなサムネイル（画像未登録時はプレースホルダー）
- **情報セクション**: 物品名、企業名、型番、個数、保管場所、備考
- **個数操作**: +/- ボタンで素早く個数変更
- **アクションボタン**: 「編集」「削除」
- **閾値設定**: アラート閾値の表示・変更

#### 物品追加画面（AddItem）
- **フォーム項目**: 物品名（必須）、企業名、型番、個数（必須）、保管場所、備考
- **画像添付**: カメラ撮影 or ライブラリから選択
- **アラート閾値**: 在庫アラートの閾値設定（デフォルト: 5）
- **保存ボタン**: 画面下部に固定配置

#### 物品編集画面（EditItem）- モーダル表示
- AddItemと同一レイアウト、既存データをプリフィル
- 「保存」ボタンで更新

#### アラート画面（Alerts）
- **アラート一覧**: 閾値を下回った物品のリスト
- **各アラート項目**: 物品名、現在の個数、閾値、「補充済み」ボタン
- **空状態**: 「在庫不足の物品はありません」メッセージ

## 4. キーユーザーフロー

### 4.1. 物品登録フロー
1. 「追加」タブをタップ
2. 物品名・個数（必須）を入力
3. 必要に応じて企業名・型番・保管場所・備考を入力
4. 画像添付ボタンをタップ → カメラ or ライブラリを選択
5. アラート閾値を設定
6. 「保存」ボタンをタップ → 在庫一覧に遷移

### 4.2. 在庫確認・更新フロー
1. 在庫一覧画面で物品カードをタップ
2. 詳細画面で現在の在庫状況を確認
3. +/- ボタンで個数を素早く調整
4. 変更は即座に保存される

### 4.3. アラート確認フロー
1. 「アラート」タブをタップ（バッジで件数表示）
2. 在庫不足の物品一覧を確認
3. 物品をタップして詳細画面へ遷移
4. 補充後、個数を更新 → アラートが自動解除

## 5. インタラクションデザイン

- **カードタップ**: opacity: 0.7 のフィードバック
- **ボタンタップ**: scale: 0.97 + Haptics.impactAsync(Light)
- **個数変更**: Haptics.impactAsync(Medium)
- **削除操作**: 確認ダイアログ表示 + Haptics.notificationAsync(Warning)
- **保存成功**: Haptics.notificationAsync(Success)

## 6. バナー広告配置

- 在庫一覧画面の最下部（タブバーの上）に固定バナー広告を表示
- 広告高さ: 50px（標準バナーサイズ）
- 広告が読み込まれない場合は非表示（レイアウトシフトなし）
